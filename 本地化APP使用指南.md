# 智能烤箱控制器 - 本地化APP使用指南

## 🎯 解决方案概述

针对Web界面频繁连接中断的问题，我们提供了两种本地化APP解决方案：

### 1. 安卓APP (推荐)
- **稳定性**: 比Web界面更稳定的TCP连接
- **响应速度**: 直接TCP通信，无需HTTP协议解析
- **资源消耗**: 减少ESP8266的内存压力

### 2. Python桌面客户端 (立即可用)
- **快速测试**: 无需安装Android开发环境
- **跨平台**: 支持Windows、macOS、Linux
- **功能完整**: 包含所有控制功能

## 📱 安卓APP使用说明

### 环境要求
- Android Studio 2020.3.1 或更高版本
- Android SDK 31 (Android 12)
- 最低Android版本: 5.0 (API 21)

### 编译和安装

1. **打开项目**
   ```bash
   # 在Android Studio中打开项目
   File → Open → 选择 SmartOvenApp 文件夹
   ```

2. **构建APK**
   ```bash
   Build → Build Bundle(s) / APK(s) → Build APK(s)
   ```

3. **安装到设备**
   - 将生成的APK文件传输到Android设备
   - 在设备上允许"未知来源"安装
   - 安装并运行APP

### 连接设置
- **IP地址**: 192.168.16.103 (设备当前IP)
- **端口**: 8080
- **协议**: TCP

### 功能说明

#### 🔌 连接控制
- **连接/断开**: 一键连接和断开设备
- **状态显示**: 实时显示连接状态

#### 🔧 设备控制
- **获取状态**: 获取当前温度、加热状态等信息
- **设置温度**: 设置目标温度 (支持快速温度按钮)
- **切换加热**: 开启/关闭加热功能
- **切换模式**: 切换PID控制/开关控制模式

#### 📊 通信日志
- **实时显示**: 所有发送和接收的命令
- **自动滚动**: 自动显示最新通信内容

## 💻 Python桌面客户端使用说明

### 环境要求
- Python 3.6 或更高版本
- 无需额外依赖 (使用标准库)

### 快速启动

1. **运行客户端**
   ```bash
   # 在项目根目录执行
   python SmartOvenDesktopApp.py
   ```

2. **连接设备**
   - 输入IP地址: 192.168.16.103
   - 输入端口: 8080
   - 点击"连接"按钮

### 功能特点

#### 🚀 快速温度设置
- 预设常用温度: 100°C, 150°C, 200°C, 250°C
- 一键设置，无需手动输入

#### 🎯 精确温度控制
- 支持自定义温度设置
- 温度范围: 0-300°C

#### 📈 实时监控
- 通信日志实时更新
- 连接状态实时显示

## 🔧 技术实现细节

### 通信协议
```
命令格式:
GET_STATUS          # 获取设备状态
SET_TARGET:温度值    # 设置目标温度 (如: SET_TARGET:200)
TOGGLE_HEAT         # 切换加热状态
TOGGLE_MODE         # 切换控制模式
```

### 响应格式
设备返回精简的状态信息，包含:
- 当前温度
- 目标温度
- 加热状态
- 控制模式
- 系统状态

## 🚀 使用建议

### 日常使用
1. **推荐使用Python桌面客户端** - 响应最快，稳定性最好
2. **移动场景使用安卓APP** - 适合手机远程控制
3. **Web界面作为备用** - 用于快速查看状态

### 故障排除
1. **连接失败**: 检查设备IP地址是否正确
2. **无响应**: 确认设备TCP服务器已启动
3. **频繁断开**: 检查网络稳定性

## 📞 技术支持

如果遇到任何问题，请检查:
1. 设备串口监视器输出
2. 网络连接状态
3. 防火墙设置

**💡 提示**: 本地化APP相比Web界面能提供更稳定、更快速的连接体验，特别适合需要频繁控制的场景。
